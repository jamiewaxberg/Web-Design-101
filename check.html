Testing Process

This document illustrate the steps that we need to follow in order to conduct a proper testing for a new implemented tracking code

Check if the tracking code is available

First of all, we need to check if the tracking code is available in our system or not. Use this tool in order to look up the code. trackingcodes.cleverbridge.com

Add a new Tracking Code in the CA

Go to: Tools -> Add Third-Party Tracking Setup

Under "General":

  • Look for the tracking code that you want to implement

Under "Parameter and Filter":

  • Make sure to select "triggers with parameter" and add a parameter name in order to test the code first. We will update it to be "trigger without parameter" once we make sure that everything works properly.
  • "Purchase status set" should be "All" for the testing step so it can trigger for test orders. After testing it, we will update it to the correct value.

Under "Parameters":

  • Use the "cleverbridge Tracking Code Documentation" in order to view how the parameters are filled out in the actual code.

Under "Sample HTML"

You can view the code and the different static and dynamic parameters available.

Test the tracking code in the browser

  1. Use the link Generator to create a test link and add "&tracking=xxxx" at the end of the URL to trigger the tracking code. Following the example:

    Client Id#677
    Tracking parameter: xxxx
    URL: https://www.cleverbridge.com/677/?scope=checkout&cart=130743&tracking=xxxx


  2. Check the page where the tracking code should trigger in the "cleverbridge Tracking Code Documentation"


    For the example, it only fires in the confirmation page so we need to complete a purchase in order to test it.
  3. View the Source Code to verify that the tracking code is firing correctly.Following the exmple:

  4. Once it is tested in every scope, go back to the Parameter and Filter section fo the tracking code to update the trigger options accordingly.

Ready to Go

Now the setup is done and the tracking can be used.
