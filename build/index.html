What are websites?

Intro, Ch. 1-4

Basics of a site

A website is a folder of files
  • An HTML file = a web page
  • A folder of HTML files = a multi-page website
  • Folder structure = site structure
    • Hierarchy of files Home page vs. sub-pages, article pages, etc.
  • We will be using Sublime Text for code documents
HTML creates content
  • HTML = Hypertext Markup Language
  • Bare-bones default style without CSS
  • Structure of a basic HTML page:
    • HTML5 doctype tag Head and body tags Meta tags
  • HTML5 is the latest edition of HTML
    • First public working draft released in 2008 Finalized and standardized in October 2014
CSS creates visual styling
  • CSS = Cascading Style Sheets
  • Defines properties like font, type size, colors
  • Positions elements on page
  • CSS3 is the 3rd level of the language, still under development
    • Has more modern properties that are not backwards-compatable in older browsers (generally IE8 and older)

Intro to tags

Tags wrap the content
  • Tag structure
    • Opening and closing tags vs. self-closing tags Attributes define properties of the tag
  • Content goes between opening and closing tags
Syntax is the "grammar" of the coding language
  • Every coding language has different syntax
  • Certain tags apply a default style to the content inside of them
    • Ex.: heading tags (<h1>, <h2> ... <h6>), font-style tags (<b>, <strong>, <i>), links (<a>)
  • Certain tags have attributes
  • HTML tags can "nest" to achieve desired markup structure
Text
  • Paragraph tags (<p>), bold tags (<b>), and span tags (<span>) are most common for putting body text onto a web page
  • Heading tags (<h1>, <h2> ... <h6>) are used for headings with respective importance
Lists
  • Ordered lists (<ol>) and unordered lists (<ul>) with list items (<li>) nested inside
Links
  • Anchor tag with href attribute (defines link destination)
    • <a href="url here">link</a>
  • Relative vs. absolute paths
    • Relative paths point at another file in your project folder, in respect to where the HTML file is Absolute paths point at the full path of something, regardless of where you're starting from

Images, tables, & forms

Ch. 5, 6, 7

Images
  • Image tag always has src=" " attribute
    • <img src="image path here"> Self-closing tag
  • Best practice to store images in a folder within your project folder and use relative paths
  • alt=" " attribute provides alternative text that can be read by a screen reader for accessability purposes
    • More on accessability and the web
  • <img> tag has attributes
    • width=" ", height=" ", and align=" " Better practice to achieve these styles with CSS
  • Images are inherently an inline element
    • Most HTML elements are block elements, but some are inline (this will be expanded on later)
  • Bitmap vs. vector (Photoshop vs. Illustrator)
    • Vector is preferable for everything that's not a photograph Vector graphics are infinitely scalable without losing resolution or quality Bitmap images will lose resolution and quality each time they are scaled
  • Image file types
    • Use JPG for photographs Use PNG for vectors and bitmap images that have fewer colors (logos, icons) and anything containing transparency Use GIF for animations
Tables
  • Used for creating excel-like tables
    • Everything gets wrapped in <table> tag <tr> tags wrap rows <th> and <td> wrap table cells Tables and cells can nest like other tags
  • Not used much anymore except for email templates
Forms
  • Inputs that send data to a server
    • <input> tags with various attributes <label> tags make labels dedicated to each input id=" " attribute on <input> and for=" " attribute on <label> with same value links the two together
  • Entire form gets wrapped in <form> tag
    • <form> tag needs action=" " and method=" " attributes to know where to send the data
  • type=" " attribute defines type of input (text, password, checkbox, radio button)
  • Attributes like name=" " send additional data to server
  • <select> is used to create dropdown menus
  • type="submit" attribute creates submit button (clicking this sends filled-in data to server)
  • Certain type attribute values (email, url) have validation services built into HTML5
    • Verifies input content against a set of criteria, throws error if content doesn't meet criteria

  <label for="firstName">First Name:</label>
  <input type="text" id="firstName">

Intro to CSS

Ch. 10, 11

Writing CSS
  • Two parts of every CSS rule: the selector and the declaration
    • The selector is the name of the HTML element and is written first The declaration goes next, inside curly brackets { }
  • The declaration is comprised of one or more definitions, which is a property followed by a value
  • When defining the same style to multiple elements, the selector can have multiple element names separated by commas
Cascading Style Sheets
  • Rules are read from top to bottom and can overwrite each other
  • Child elements inherit styles from their respective parent elements
  • CSS rules cannot nest
  • It is best practice to put style resets at the top of your style sheet
  • The right-hand column of a modern browsers' inspector is where the CSS is
    • Has the ability to uncheck/recheck styles and write in new styles
Attaching your style sheet to your HTML document
  • Use the <link> tag to link the style sheet to the document
  • <link> needs three attributes to properly attach your style sheet:
    • The href=" " attribute provides the relative path to the style sheet The type="text/css" and rel="stylesheet" attributes tell the rendering engine that this is a CSS style sheet
  • Styles can also be written in <style></style> tags in the head of the document, but it's better practice to link an external stylesheet
  • Styles can be written inline with the HTML, using the style=" " attribute on an element, but this method is used pretty rarely
Colors
  • color: ; defines type color, and background-color: ; defines background color
  • Color names
    • 147 predefined color names recognizable to browsers
  • Hexadecimal codes
    • Prefered method # followed by 6 letter or number characters Hex values that are 6 of the same character can be written with # and three characters (#000 is the same as to #000000)
  • RBG values
    • rgb(x, y, z) where x represents amount of red, y represents about of green, and z represents amount of blue, and these variables are a value 0-255 CSS3 supports RBGA, where there is a fourth value 0-1 declaring opacity (with 0 being fully transparent and 1 being fully opaque)

Type, lists, tables, and forms

Ch. 12, 14

Type lingo
  • Serif fonts have outward lines and details at the end of each stroke, and are generally more traditional looking
  • Sans serif fonts have straight ends to the strokes, and are generally more modern looking
  • X-height is the height of the lowercase x
  • The baseline is the line the letters sit on
  • Cap height is the height of the capital letters
  • Ascenders are the upward strokes than extend above the cap height
  • Descenders are the downward strokes that descend below the baseline
  • Kerning is the horizontal space between letters
  • Leading is the vertical space between lines
Type for the web
  • General rule of thumb is to never include more than three fonts in any design, whether it's web or print
    • Two fonts is usually sufficient; only use three if the design really needs it Generally, sans-serif and serif fonts pair well together Sans serif fonts for body copy are more readable on the web
  • font-family: ; definition
    • Value is a list of fonts separated by commas and spaces When a font name is multiple words, put it in quotes First font is tried, and if it can't be loaded, the rendering engine goes down the list Always include a basic font toward the end, such as Arial or Times New Roman, followed by either the generic serif or sans-serif at the end
  • Web fonts need to be used when you want a custom font that is not web-safe
    • Web safe font combinations Use @font-face declaration to load different files on the font into your website These font files would live in your project folder in a "fonts" folder, similarly to images Alternative to @font-face is Google Fonts

  @font-face {
    font-family: 'myWebFont';
    src: url('myWebFont.eot');
    src: url('myWebFont.eot?#iefix') 
    format('embedded-opentype'), 
    url('myWebFont.woff') format('woff'), 
    url('myWebFont.ttf') format('truetype'), 
    url('myWebFont.svg#latobold') format('svg');
    font-weight: normal;
    font-style: normal;
  }

Type continued & more advanced CSS selectors

Ch. 10 & 12

Type cont.
  • Three methods of type size units:
    • Pixels (px) is the most basic method, similar to setting type size in Microsoft Word, and gives you most control over the size Percentages start as a fraction of the default browser type size (16px), and are always relative to the parent container's type size Ems are also relative, similar to percentages. 1em is equivalent to the width of a 16px "m" in the given type face
  • CSS properties like font-weight, font-style, text-transform, and text-decoration format text
  • CSS properties like text-align and vertical-align align text
  • Styling links with psuedo-classes
    • Attaching :hover, :visited, :active, and :focus to the end of the selector style different "states" of the link
CSS selectors with classes and IDs
  • Classes and IDs can be their own selectors, or they can be attached to the HTML element name
    • . preceeds class names and # preceeds ID names when writing selectors When writing a class or ID in addition to the tag name, it would be written as tag.className or tag#idName
  • Since ID names are supposed to identify one element in the entire HTML document, it is less common that you would need to write the tag name in addition to the ID
  • Since class names can be repeated throughout your markup and put on different element types, you do often need to write the tag name in addition to the class name to get more specific with your selectors
  • When writing multiple class names of a single element as a selector, write them in succession with no spaces in between, i.e. .className1.className2
Advanced CSS selectors
  • These are used for getting more specific with selectors
  • See the full list of advanced selectors here
    • These selectors use special characters between multiple element names Also for reference, p. 238 of the book has a more simplified list of these advanced selectors, containing the ones which are most commonly used
  • Psuedo classes are used for styling specific states or types of elements
    • These go directly after the element name, with a : and the name of the psuedo class

  p.heading {
    color: blue;
    font-size: 21px;
    font-weight: bold;
  }

The box model & layout

Ch. 13 & 15

The key to CSS: the box model

Every HTML element has a box around it
  • Margin is white space around the outside of the box
  • Padding is white space between the edge of the content and the edge of the box
    • Both margin and padding can be written in two ways: defining one side, or defining all 4 sides margin-top: 10px; vs. margin: 10px;
  • width: ; and height: ; properties can control size of the box
    • Values of these properties can be a set px value or a percentage max-width: ; is friendlier to responsive design, often accompanied by width: 100%; min-width: ; can set minimum width of the box
  • overflow: ; property can put a scroll bar in the box
  • Elements can be centered within their container using margin: 0 auto;
  • Elements can be styled as a box
    • border: ; can style the box itself and needs 3 values: width, style, and color box-shadow: ; can give a box a shadow, and needs several values border-radius: ; rounds the corners of an element box-shadow and border-radius need several vendor prefixes for optimal cross-browser compatability
Understanding the document flow
  • The normal document flow is the natural arrangement of these boxes from top to bottom and left to right
  • Elements can participate in this flow or be "taken out of" the normal flow and positioned alternatively
  • Use the display: ; property to change default structure of an element
    • display: block; and display: inline-block; keep elements within the normal flow, but inline-block allows the next element to go directly next to the current element without breaking to the next line dispaly: inline; takes elements out of the normal flow but keeps them in place display: none; hides elements

The box model continued & images

Ch. 15 & 16

Understanding the document flow cont.
  • The float and position properties take elements out of the flow and position them elsewhere
  • float has three values: left, right, and none
    • clear is used when you want to float an element but you don't want any elements to float next to it and has three values: left, right, and both
  • position is used when you want the element to always appear in the same spot, relative to a container or the document itself
    • position: absolute; positions an element within the closest positioned ancestor element (if there is no positioned ancestor, this property will position the element relative to the browser window) position: relative; will position the element relative to itself position: fixed; will position the element relative to the browser window position: static; is the default
  • z-index: ; is used for overlapping elements and take a numeric value with no unit
    • Higher number = higher layer Element needs a position definition in order for z-index to work
Images
  • width: ; and height: ; properties can control the size of images
  • When including images to accompany body copy text, images can be floated to the side of a text area or paragraph to achieve a desired layout/design
  • When creating background images, there are three properties needed: background-image, background-repeat, and background-position
    • background-image defines the path to the image with the url(" "); value background-repeat determines if the image will repeat/tile or not; repeat, repeat-x, and repeat-y values are used for tiling the image, and no-repeat is used when the image should appear only once background-position takes a top/bottom value and a left/right value to determine the position of the image, written as top 10px left 10px; when values are both 0, the value can be written as 0 0 without directional labels All three of these definitions can be written as a single background property
  • With CSS3, the background-image property can be leveraged for creating gradients

  .wrapper {
    background: url("assets/backgroundImage.jpg") 
    no-repeat 0 0;
  }

Grid structures & HTML5 layout

Ch. 15 & 17

Grids
  • Grids help structure composition, create consistency, and create visually pleasing and balanced designs
  • All grids are based off of a wrapper width
    • Common widths are 960px, 990px, 1170px
  • 12-column grids are very common because they are divisible into sixths, quarters, thirds, and halves horizontally
    • 960.gs Bootstrap
  • Elements of a row are all floated left side-by-side with set widths and a consistent amount of margin on each side where appropriate
  • Using these grids requires an understanding of their class name conventions
    • Can be a down side, as grid class names are often descriptive of the page structure and not the content To combat this issue, it can be helpful to use the grid style sheet in addition to your own style sheet, so that you can use both the grid class names and also your own descriptive class names When using multiple style sheets, you can use @import in the single style sheet that's linked into the HTML, or you can link multiple style sheets into the HTML
  • These grids are readily responsive and can help create a design that looks great on all devices
  • While grids can be helpful and can speed up the workflow of a project, you should understand what the grids are doing and how you can achieve the same outcome with your own handwritten CSS
HTML5 Layout
  • Use the tags as descriptors for your content
  • Arrange the content sections using common conventions of the web
  • Use HTML5 tags in combination with a grid to achieve a highly structured design with strong markup
  • Write a fallback rule in the CSS that tells older browsers what to do with these tags
  • Use Google's HTML5 shiv javascript to make these tags work in browsers older than IE9

Process & design, practical info

Ch. 18 & 19

Websites are for users
  • When making sites and/or site designs, have a user-centric mindset
  • Identify who the target audience is
    • Ask yourself questions about the end users' demographic, region, occupation, etc. to gain a better understanding of the user The tagret audience is not always individual people - it can also be companies or other entities
  • What is the user trying to accomplish in visiting your site?
    • Keep this in mind, and design & build the site to foster the accomplishment of these tasks and/or goals
  • How frequently will the site be used, and how long will it live in the web?
    • It is best practice to make updates to widely-used sites on a schedule instead of a case-by-case basis If the site is going to have a long lifespan, content will likely need to be updated periodically
Designing the site structure
  • Site maps help organize the pages of a site
    • These are flow charts that organize pages of a site based on content When making decisions about the organization, keep the users' end goals in mind Determine what information needs to go on each page, and if it needs to appear on multiple pages
  • Creating wireframes can help you sketch out the structure of information on each page
  • Create a visual hierarchy that will lead the user to different visual elements in order of importance
  • Group visual elements according to the six gestalt principles: similarity, continuation, closure, proximity, figure/ground, and symmetry & order
Practical info
  • SEO (search engine optimization) helps your sites rank higher in search engines
    • Search engines read your site code an content and look for keywords matching or relevant to the search term There are seven key areas of the page the browser looks for these keywords, listed in order of importance: the page title, the URL/web address, headings, text, link text, image alt text, and page descriptions (meta tags) Other sites linking to your site will also improve its search engine ranking
  • Collecting analytics data can help you see how many visits and page views your site is getting, as well as some more specific data (unique visits, how long visitors are spending on each page)
    • Google Analytics
  • Buying a domain
    • First, check if the domain is available Next, buy the domain from a hosting company such as Bluehost, 1&1, GoDaddy
  • Once you buy the domain, download an FTP client such as FileZilla
    • Connect to the domain's server (there should be instructions provided) with the FTP program and upload your outer site folder

Advanced topics, technologies, & resources

Designing responsively
  • Responsive design is what makes web pages compatible on all screen sizes
    • The three general screen sizes/devices to be aware of are desktop, tablet, and mobile
  • Use media queries to change and add CSS rules at different break points
    • Use @media screen and () with a min-width [value] or max-width [value] property inside the parenthasis The media queries execute the contained code when the min-width or max-width is reached
  • Generally, your default styles will be for desktop, and you will have media queries overwriting and adding styles for tablet and mobile
    • For tablet queries, use (max-width: 768px) { } For mobile queries, use (max-width: 480px) { } For desktop queries, use (min-width: 769px) { }
More advanced topics
  • CSS preprocessors: Sass (syntactically awesome style sheets)
    • Extends the functionality of the CSS language: allows for styles properties to nest, variables, mixins, and extends Requires configuring a compiler with Ruby or Node.js Styles are split into separate, organized files (partials) with the extention .scss, where the file name starts with _ Partials are imported to a single .scss file that has no _ at the beginning of the file name This file is compiled into regular CSS in a separate file, and this file is the style sheet you link to your site
  • JavaScript and jQuery
    • The JavaScript language as a whole controls the interactivity of websites jQuery is a JavaScript library with its own properties, developed specifically for the purpose of manipulating the DOM tree (document object model) Creates the ability to easily move HTML elements, add/remove classes, create events on objects, add styles, and more
